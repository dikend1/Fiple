# Fiple — Дизайн-спека (MVP)

**Дата:** 2026-06-18
**Статус:** Утверждён для перехода к плану реализации

---

## Что такое Fiple

Fiple превращает iPhone в **пульт управления Mac**. Вместо постоянного
переключения между приложениями, вкладками, Dock и Spotlight пользователь
одним нажатием с телефона запускает приложения, открывает сайты и
восстанавливает целые рабочие сценарии на своём Mac.

**Главная идея — не запуск приложений, а восстановление контекста.**
Чтобы после любого отвлечения вернуться в состояние потока одним нажатием.

## Проблема

Разработчики, дизайнеры, студенты и фаундеры работают сразу в нескольких
инструментах (Cursor, ChatGPT, GitHub, Terminal, документация, браузер).
После каждого отвлечения приходится заново открывать приложения, искать
вкладки и восстанавливать контекст — это съедает время и разрушает
концентрацию. Spotlight, Dock, Raycast, Alfred помогают запускать
приложения, но не помогают быстро вернуться к рабочему состоянию.

## Решение

Пользователь создаёт на Mac **плитки (Tiles)**. Плитка запускает одно или
несколько действий. Плитка с несколькими действиями = рабочее пространство
(workspace preset), например «Start Coding», «Deep Work», «Ship Update».
Одним нажатием с телефона восстанавливается вся рабочая среда.

---

## Архитектура

Два нативных приложения (**SwiftUI**) + локальный протокол поверх Wi-Fi.

```
┌─────────────────┐         Wi-Fi (LAN)          ┌─────────────────┐
│  iPhone (пульт) │  ◄──── WebSocket ──────►     │  Mac (компаньон)│
│   SwiftUI       │    Bonjour discovery         │   SwiftUI       │
│                 │    + код для пары            │   menu bar app  │
│ • видит плитки  │                              │ • хранит плитки │
│ • жмёт → запуск │                              │ • создаёт/правит│
│                 │                              │ • исполняет     │
└─────────────────┘                              └─────────────────┘
                                                 open -a / open URL / open file
```

### Роли

- **Mac — источник правды + управление.** Хранит плитки, создаёт /
  редактирует / сортирует / удаляет их в своём UI, знает установленные
  приложения, исполняет действия.
- **iPhone — только пульт.** Видит список плиток (получает с Mac),
  нажимает → запускает. Ничего не редактирует. При изменении плиток на Mac
  список на телефоне обновляется автоматически.

### Стек

- macOS-компаньон: SwiftUI, приложение в меню-баре.
- iOS-пульт: SwiftUI.
- Транспорт: Bonjour/mDNS (обнаружение в фоне) + WebSocket (команды).
- Только локальная сеть. Без аккаунтов, паролей, облака.

## Подключение (без списка устройств)

Принцип: **никакого списка «устройств рядом»** — только ввод кода.

1. На Mac в меню-баре нажимаешь «Подключить» → показывается **4-значный код**.
2. На iPhone — одно поле ввода кода.
3. Телефон **тихо в фоне** через Bonjour находит Fiple-Mac'и в той же
   Wi-Fi сети; код выбирает нужный Mac + проверяет подлинность.
4. После успеха пара **сохраняется**: телефон помнит Mac и при следующем
   запуске соединяется автоматически.
5. Кнопка «Отключиться» разрывает пару; при повторном подключении Mac
   показывает **новый код**.

**Ограничение MVP:** оба устройства должны быть в одной Wi-Fi сети.
Соединение настраивается один раз.

## Модель данных

```
Tile
 ├─ id, name, icon, color, order
 └─ actions: [Action]        // одно действие или несколько = workspace

Action (тип + параметр)
 ├─ launchApp   → bundleId / путь приложения
 ├─ openURL     → ссылка
 └─ openFile    → путь к файлу/папке (открыть в дефолтном/указанном приложении)
```

Отдельной сущности «workspace» нет — это плитка с несколькими `actions`,
которые Mac выполняет по очереди. Меньше сущностей — меньше сложности.

---

## Поток данных (нажатие плитки)

```
iPhone: тап по «Start Coding»
   → WS-сообщение { run: tileId }
Mac: находит плитку → по очереди исполняет actions
   → шлёт обратно статус { tileId, ok/err, per-action result }
iPhone: показывает результат (галочка / что именно не открылось)
```

---

## Управление плитками

- Вся работа с плитками — **в приложении на Mac**: создать, изменить,
  перетащить порядок, удалить; выбрать приложение / URL / файл из реальных
  на этом Mac.
- iPhone получает готовый список плиток и только **отображает и запускает**.
- На телефоне нет форм редактирования и пикеров — только сетка плиток,
  статус подключения и обратная связь по запуску.

---

## Обработка ошибок

- **Связь пропала** → телефон показывает «Не подключено» и кнопку
  переподключиться; команды не отправляются вслепую.
- **Приложение не установлено / файл не найден** → Mac возвращает ошибку
  по конкретному action; телефон подсвечивает, что именно не сработало.
  Остальные действия workspace всё равно выполняются.
- **Код неверный / просрочен** → понятное сообщение, код можно
  перегенерировать.

## Границы MVP

**Входит в MVP:**

- Пара по коду (локально, одна Wi-Fi, сохраняется).
- Плитки с действиями: launch app / open URL / open file.
- Workspace-пресеты (плитка с несколькими действиями).
- Создание / редактирование / сортировка / удаление плиток **на Mac**.
- Список установленных приложений на Mac для выбора при создании плитки.
- Статусы выполнения действий на телефоне.

**НЕ входит (после MVP):**

- **AI-голос (v1.1):** голосовая команда → действие. Включая умный разбор
  намерения («поставь Bad Habits» → открыть Spotify/YouTube/Яндекс Музыку
  и найти трек).
- Интеграция с Apple Shortcuts.
- Сложные многошаговые сценарии.
- Расстановка/позиционирование окон (Accessibility API).
- Синхронизация между устройствами.
- Совместные рабочие пространства для команд.
- Аналитика продуктивности.

---

## Видение

Самый быстрый способ вернуться к работе. Одно нажатие → мгновенный
контекст → сразу в работу. Меньше переключений, меньше трения, больше
концентрации.

После MVP Fiple развивается в платформу автоматизации рабочего
пространства (см. «НЕ входит в MVP» как дорожную карту).
