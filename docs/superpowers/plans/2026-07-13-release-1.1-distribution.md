# Fiple 1.1 Release — Distribution Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Make the repo produce three release artifacts from one branch — iOS 1.1 (App Store, with terminal client), Mac 1.1 MAS (sandboxed, no terminal), Mac 1.1 Developer ID (notarized site download with terminal) — with build postures encoded in `project.yml` instead of hand-flipping.

**Architecture:** Three named Xcode configurations (`Debug` / `Release-MAS` / `Release-DevID`) with per-config settings on the FipleMac target, plus explicit archive schemes. iOS gains a terminal explainer sheet for users whose Mac can't host a shell, linking to `fiple.app/download`.

**Tech Stack:** XcodeGen (`project.yml`), Swift 6 / SwiftUI, swift-testing (FipleKit), `xcodebuild` + `notarytool` for the release runbook.

Spec: `docs/superpowers/specs/2026-07-13-release-1.1-distribution-design.md`.

## Global Constraints

- Version for this release: `MARKETING_VERSION 1.1.0`, `CURRENT_PROJECT_VERSION 6`.
- Teams: `M3UQXL2W5B` = dev (device runs, Debug only); `VXJ8BY6538` = distribution (MAS + Developer ID). Never archive Debug.
- The MAS Mac posture is exactly: `ENABLE_APP_SANDBOX YES` + `Apps/FipleMac/Fiple.entitlements`.
- The DevID posture is: sandbox off + `ENABLE_HARDENED_RUNTIME YES` (required for notarization).
- Existing CLAUDE.md verify commands (`xcodebuild -scheme FipleMac … build`, `-scheme FipleiOS … build`, `cd FipleKit && swift test`) must keep working unchanged.
- No new third-party dependencies.
- Deviation from spec (approved rationale): scheme names are `FipleMac` (archives Release-MAS — the safe default) and `FipleMac-DevID`; no separate `FipleMac-MAS` scheme is needed.

4. **[assistant] DevID archive + notarize + staple** (needs an App Store Connect API key or app-specific password once — store as `notarytool` keychain profile `fiple-notary`):
   ```bash
   xcodebuild -project Fiple.xcodeproj -scheme FipleMac-DevID archive -archivePath build/FipleMac-DevID.xcarchive
   # ExportOptions-DevID.plist: {method: developer-id, teamID: VXJ8BY6538}
   xcodebuild -exportArchive -archivePath build/FipleMac-DevID.xcarchive \
     -exportOptionsPlist ExportOptions-DevID.plist -exportPath build/devid
   ditto -c -k --keepParent build/devid/Fiple.app build/Fiple-1.1.0-mac.zip
   xcrun notarytool submit build/Fiple-1.1.0-mac.zip --keychain-profile fiple-notary --wait
   xcrun stapler staple build/devid/Fiple.app
   ditto -c -k --keepParent build/devid/Fiple.app build/Fiple-1.1.0-mac.zip  # re-zip stapled app
   spctl -a -vv build/devid/Fiple.app  # expect: accepted, source=Notarized Developer ID
   ```
